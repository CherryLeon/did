# 论文修改专家规范

## 角色定义

论文修改专家（/thesis-revision）是根据金融专业硕士导师的评审意见，专门负责修改论文、代码与数据的角色。其核心任务是确保论文描述与代码数据的一致性，解决评审中发现的各类问题。

## 工作流规范

论文修改专家在执行任务时，必须严格遵循以下工作流程：

### 工作流程图

```
步骤1：读取论文文件 → 步骤2：调用金融导师评审 → 步骤3：生成点评文件
                                                    ↓
步骤7：质量检查 ← 步骤6：生成差异报告 ← 步骤5：修改论文/代码/数据
                                                    ↓
                                            步骤4：调用论文专家
```

### 步骤详细说明

#### 步骤1：读取论文文件
- 读取论文文件（通常是 `论文总结-DS模拟结果.md`）
- 分析论文的实证部分结构
- 识别关键数据、模型设定和结果描述

#### 步骤2：调用金融导师进行评审
- 触发命令：`/thesis-review`
- 金融专业硕士导师角色评估论文是否满足硕士论文学术规范要求
- 重点审查：数据一致性、表格规范性、实证结果规范性、经济价值测算、论文结构、细节规范性
- **输出**：评审意见（问题描述、修改建议）、修改优先级建议

#### 步骤3：生成点评文件
- **文件命名**：`comments_v{版本号}.md`（首次v1，后续v2、v3...）
- **存放位置**：`d:\trae_project\did\comments_v{版本号}.md`
- **内容**：总体评价、具体问题分析、修改优先级建议、总体建议

#### 步骤4：调用论文专家进行修改
- 触发命令：`/thesis-revision`
- 根据评审意见修改论文、代码与数据
- 按照优先级处理问题

#### 步骤5：修改论文/代码/数据
- **涉及文件**（共3个，必须全部修改）：
  1. `d:\trae_project\did\论文总结-DS模拟结果.md` - 论文内容
  2. `d:\trae_project\did\test_data_generator.py` - 数据生成代码
  3. `d:\trae_project\did\empirical_analysis.py` - 实证分析代码

- **高优先级修改**（必须修改）：
  1. 观测值数量统一：确保全文数据一致性
  2. 时间窗口统一：与数据生成规范保持一致

- **中优先级修改**（建议修改）：
  3. 表格顺序调整：使与文字叙述顺序一致
  4. 基准期数据核实：确认t-5数据缺失原因
  5. 动态效应期数说明：考虑是否需要那么多期
  6. 安慰剂检验完整性：引用分布图，提供更多细节

- **低优先级修改**（可选修改）：
  7. 章节结构调整：将结论部分独立成章
  8. 术语统一：建立术语表
  9. 参考文献引用：确保完整收录

## 数据一致性规范

### 1. 核心数据规格

根据论文要求，实证分析必须严格遵循以下数据规格：

```
分行数量：86家（处理组47家 + 对照组39家）
客户总数：514,601个
时间窗口：2022年3月至2025年3月（共37个月）
观测值数量：
  - 客户-月度面板：514,601 × 37 = 19,040,437（原始）
  - 分行-月度面板：86 × 37 = 3,182
  - PSM样本：514,601
  - 基准DID回归：86 × 37 = 3,182（分行层面）或 514,601（客户层面）
```

### 2. 数据层级定义

论文必须明确说明实证分析使用的数据层级：

| 数据层级 | 观测值数量 | 适用场景 |
|---------|-----------|---------|
| 客户-月度面板 | 514,601客户 × 37月 = 19,040,437 | 描述性统计、分场景回归 |
| 分行-月度面板 | 86分行 × 37月 = 3,182 | 基准DID回归（可选聚合） |
| PSM样本 | 514,601 | 倾向得分匹配分析 |

### 3. 表格观测值规范

所有表格中的观测值数量必须与实际使用的样本量一致：

- **表4-2（DID变量定义）**：观测期数应为37个月
- **表4-9（基准DID回归）**：观测值应为3,182（分行层面）或19,040,437（客户层面）
- **平行趋势检验**：dynamic_-11到dynamic_23对应37个月窗口

### 4. 时间窗口规范

政策实施时间为2023年3月，时间窗口划分如下：

```
政策前：2022年3月至2023年2月（12个月）
政策后：2023年3月至2025年3月（24个月）
总计：37个月
```

事件研究法中：
- 相对时间t=-11对应2022年4月
- 相对时间t=-1对应2023年2月（基准期）
- 相对时间t=0对应2023年3月（政策实施当月）
- 相对时间t=23对应2025年3月

建议将第四章拆分为三个独立章节：

```

  第一节 PSM匹配与平行趋势检验

  第二节 基准回归与分场景分析
    - 基准DID回归结果

第六章 经济价值与研究结论
```

```
表5-2：动态效应分析结果
```

### 3. 表格注释规范

所有表格必须包含以下注释要素：

```markdown

    标准误在分行层面进行聚类；数据来源：[具体来源]
```

### 4. 经济价值测算规范

由于无法从DID系数精确推导经济价值，建议采用以下策略：

**方案A（推荐）：定性描述**
```
删除具体数字（15.7亿、ROI超300%等），
改为定性描述：
"生成式AI技术的部署显著提升了招商银行的零售客户经营成效，
主要体现在客服运营和信贷业务场景的效率提升，以及营销经营
和财富管理场景的精准度改善。"
```

**方案B：补充计算假设**
```
如需保留具体数字，必须补充完整计算过程：
1. 明确AI系统总投入成本（研发+部署+运维）
2. 明确收益计算口径（直接收入增加 vs 成本节约）
3. 提供敏感性分析
4. 说明数据来源
```
